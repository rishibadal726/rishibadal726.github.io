Roles:

(Software) Development Engineer:

A Software Development Engineer designs, programs, builds, deploys and maintain software by using many different skills and tools in order to accomplish something. Additionally, they help build software systems that power networks and devices and ensure those systems remain functional for other people to use.
Suitability: I feel like I could be a Software Development Engineer since I do like building new stuff,but some challenges I expect to encounter are the code might not work all the time and there could be bugs in the code that prevent the program from running.

QA Engineer:

QA Engineer stands for Quality Assurance Engineer and their job is to design test plans, scenarios, scripts, or procedures. They document software defects, using a bug tracking system, and report defects to software developers.
Suitability: I feel like I could be a QA Engineer since I do like to test new stuff and see if it works or not, but some challeneges I expect to encounter are that the test fails and doesn't pass the scenario.

Project Manager:

A Project Manager deals with the process for leading the work of a team to achieve all project goals within the given constraints. This information usually given and created at the beginning of the development process. The primary constraints a project manager sets for their team are scope, time, and budget.
Suitability: I feel like I could be a Project Manager since I do like to create boundaries and constraints for a team in order to get things done for the project, but some chlleneges I expect to encounter is some team members might not meet up to the expectations or they won;t get their work done which can lead to delays.

Product Manager:

A Product Manager deals with leading the business process of planning, developing, launching, and managing a product or service. This includes the whole lifecycle of a product, from ideation to development to go and be available in the market.
Suitability: I feel like I could be a Product Manager since I do like leading the process to develop, plan, launch, and manage a product or service, but some challeneges I expect to encounter is maybe the first idea won't always work, so I should be prepared to use a different idea and be open to new suggestions to improve the product or service.

Release Engineer:

A Release Engineer works with software development engineers to state all the steps needed to release the software. Starting from how the software is stored in the source code repository, to build rules for compilation, to how testing, packagaing, and deployment are conducted in order to make the software a finished product.
Suitability: I feel like I could be a Release Engineer since I do like helping release new software and I could help with testing, packaging, and deploying the softwaare to a finished product, but some challeneges I expect to encounter is the product might not be completly ready when I check it before releasing it, so there would need to be additional changes to the software before releasing it to a new product or service.
